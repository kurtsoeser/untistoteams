Werkzeug: SharePoint – Mandanten-Freigaben

Orientierung am SharePoint Admin Center (Teilen): Was Graph in sharepointSettings liefert, in Klartext – und dort bearbeiten, wo Microsoft dieselbe Logik anbindet.

Hilfe

Dashboard
Berechtigungen: SharePointTenantSettings.Read.All / ReadWrite.All – angemeldeter Benutzer laut Microsoft SharePoint Administrator oder Global Administrator.

Aktueller Stand in Klartext

Kurz erklärt, was die geladenen Werte für SharePoint und OneDrive praktisch bedeuten.

Externes Teilen

Entspricht der Stufe „Wie weit nach außen?“ im Admin Center (für SharePoint und OneDrive gemeinsam über sharingCapability). Klicken Sie eine Stufe – unten wird das technische Dropdown mitgeführt.

Domain-Einschränkungen

Zusätzlich zur Stufe oben: welche E-Mail-Domains bei externem Teilen erlaubt oder blockiert sind (wie „Weitere externe Freigabeeinstellungen“ im Admin Center).

Die Domain-Listen greifen nur zusammen mit allowList bzw. blockList.

Gast- und Weitergabe-Verhalten

Optionen, die Microsoft ebenfalls in den erweiterten Freigabe-Einstellungen bündelt.

Wenn aus: Gäste können Inhalte, die sie nicht besitzen, nicht weiter freigeben.

Erhöht die Sicherheit bei Einladungen an eine bestimmte E-Mail-Adresse.

Datei- und Ordnerlinks

Welche Standardberechtigung (Lesen vs. Bearbeiten) und welcher Link-Typ beim Teilen vorgewählt ist – so wie im Screenshot des SharePoint Admin Centers.

Sie übersehen nichts im Graph-Teil: GET/PATCH …/admin/sharepoint/settings liefert dafür keine Properties (weder in v1.0 noch in der aktuellen beta-Doku). Online im Tool speicherbar sind weiter nur die Blöcke oben (Graph). Für Standard-Link und Standardberechtigung nutzen Sie das Admin Center oder das generierte PowerShell unten (Set-SPOTenant).

Standardberechtigung für Freigabelinks

Entspricht „Wählen Sie die Berechtigung aus, die standardmäßig für Freigabelinks ausgewählt ist“ – Parameter -DefaultLinkPermission.

Standardtyp des Links (Vorauswahl)

Welcher Radio-Button beim Teilen vorgewählt ist – Parameter -DefaultSharingLinkType.

PowerShell zum Anwenden

Tragen Sie Ihren SharePoint-Host ein (ohne https://) oder holen Sie ihn per Graph. Anschließend Skript in der SharePoint Online Management Shell oder Windows PowerShell mit Modul ausführen.
